# Phase 1 ‚Äî UI Design Contract

> Visual and interaction contract for Phase 1: Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope reminder (from CONTEXT.md D-21/D-22/D-23):** Phase 1 ships only the layout shell, design tokens, a minimal "Coming Soon" home page, an auth modal, and a stub `/account` route. Real brand content ships in Phase 2; real loyalty dashboard ships in Phase 3. This contract covers ONLY what Phase 1 renders ‚Äî but the tokens it establishes are the foundation all future phases consume.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (SvelteKit + TailwindCSS v4 CSS-first `@theme`) |
| Preset | not applicable ‚Äî shadcn is React-only; this project uses Svelte 5 |
| Component library | none (hand-authored Svelte components in `src/lib/components/`) |
| Icon library | `lucide-svelte` (open source, tree-shakeable, matches minimal editorial aesthetic; defer install until first icon needed) |

**Rationale:** RESEARCH.md ¬ßPattern 2 establishes TailwindCSS v4 CSS-first `@theme` block as the token mechanism. CONTEXT.md D-08 mandates self-hosted, preloaded, subset fonts. Fraunces + Inter are the open-source stand-ins for Canela/GT Super that RESEARCH.md ¬ßPattern 2 font licensing note explicitly recommends. Commercial fonts (Canela, GT Super) can be swapped in later without touching the token layer.

**Design system state:** Greenfield. No `components.json`, no `tailwind.config.*`, no existing tokens. This document IS the design system for Phase 1 onward.

## Spacing Scale

Declared values (all multiples of 4). TailwindCSS v4 base `--spacing: 0.25rem` (4px) generates every step as a utility: `p-1` = 4px, `p-2` = 8px, `p-16` = 64px, `p-24` = 96px.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `*-1` | Icon gaps, hairline offsets |
| sm | 8px | `*-2` | Compact element spacing, input inner padding |
| md | 16px | `*-4` | Default element spacing, body paragraph gap |
| lg | 24px | `*-6` | Card padding, form field stack |
| xl | 32px | `*-8` | Component block gaps |
| 2xl | 48px | `*-12` | Layout gaps between sub-sections |
| 3xl | 64px | `*-16` | Major section breaks |
| 4xl | 96px | `*-24` | Editorial section spacing (Aesop/Kinfolk feel ‚Äî D-09, D-10, D-11) |
| 5xl | 128px | `*-32` | Page-level breathing room for hero / Coming Soon centering |

**Exceptions:**
- Touch targets: all interactive elements (nav links, hamburger button, auth submit) MUST have a minimum 44√ó44px hit area (per WCAG 2.1 AA / iOS HIG). Implement via `min-h-11 min-w-11` utility combinations. This is a11y-driven, not a spacing token override.
- Safe area insets: `env(safe-area-inset-*)` via `--safe-top`, `--safe-right`, `--safe-bottom`, `--safe-left` CSS variables (RESEARCH.md Pitfall #5). NOT multiples of 4 ‚Äî they are device-driven. Applied only on the outermost `<SafeArea>` wrapper.

**Section-gap discipline (D-09, D-10, D-11):** Phase 1 "Coming Soon" page uses `py-24` (96px) minimum vertical rhythm between stacked elements. Editorial pacing is a first-class contract ‚Äî do not collapse to `py-4` for any reason in Phase 1.

## Typography

Four roles, two weights, two families. All sizes are rem-based so they scale with user settings.

| Role | Size | Weight | Line Height | Family | Tailwind utility |
|------|------|--------|-------------|--------|-----------------|
| Body | 16px (1rem) | 400 (regular) | 1.6 | sans (Inter) | `text-base font-normal leading-relaxed` |
| Label | 13px (0.8125rem) | 400 (regular) | 1.4 | sans (Inter) | `text-[13px] font-normal tracking-wide uppercase` |
| Heading | 28px (1.75rem) | 400 (regular) | 1.2 | serif (Fraunces) | `text-[28px] font-normal leading-tight` |
| Display | 56px (3.5rem) | 400 (regular) | 1.05 | serif (Fraunces) | `text-[56px] font-normal leading-none` |

**Weights used: exactly 2** ‚Äî `400` on both serif and sans. Editorial serifs are set at their natural (lighter) weight; sans is set at regular. No bold, no black, no italic-as-structure. If emphasis is needed within body text, use italic (serif) or tracking/uppercase (sans Label style).

**Line-height rationale:**
- Body 1.6 ‚Äî generous reading rhythm, matches Kinfolk editorial density reference (D-07, D-10).
- Heading 1.2 ‚Äî standard editorial heading compression.
- Display 1.05 ‚Äî allows the "Rule 257" wordmark / hero "Coming Soon" serif to fill space without double-spacing.

**Font stacks (from RESEARCH.md Pattern 2):**
```
--font-serif: "Fraunces", Georgia, "Times New Roman", serif;
--font-sans: "Inter", ui-sans-serif, system-ui, -apple-system, "Segoe UI", sans-serif;
```
Preloaded from `app.html` via `<link rel="preload" as="font" type="font/woff2" crossorigin>` (one woff2 per family, Latin subset only).

**What Phase 1 actually renders (typography surface):**
1. "Rule 257" wordmark in `SiteHeader` ‚Äî **Display** (serif)
2. "Coming Soon" on home `+page.svelte` ‚Äî **Display** (serif)
3. Tagline underneath Coming Soon ‚Äî **Body** (sans)
4. Nav links in header ‚Äî **Label** (sans, uppercase, tracked)
5. Hamburger menu items ‚Äî **Heading** (serif, stacked)
6. Footer placeholder columns ‚Äî **Label** (sans) + **Body** (sans)
7. Auth modal heading ("Sign in") ‚Äî **Heading** (serif)
8. Auth modal body copy ‚Äî **Body** (sans)
9. Auth modal email input ‚Äî **Body** (sans)
10. "Welcome, {email}" on `/account` stub ‚Äî **Heading** (serif)

## Color

**CONTEXT.md D-01 explicitly rejects an accent color.** The 60/30/10 rule is honored by interpreting the palette as **paper (60%) / ink-muted (30%) / ink (10%)** ‚Äî the darkest value IS the accent and is reserved for calls-to-action and structural emphasis. Photography and art on the walls (Phase 2 brand portfolio content) carries all chromatic color.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-paper` | `#fafafa` | Page background, surfaces, modal bodies, entire global baseline |
| Secondary (30%) | `--color-paper-muted` | `#eaeaea` | Input field backgrounds, disabled states, subtle card surfaces, footer top divider fill |
| Secondary hairlines | `--color-hairline` | `#e5e5e5` | 1px dividers, nav bottom border, footer column rules, input borders |
| Accent (10%) | `--color-ink` | `#0a0a0a` | Primary CTA buttons, text headings, body copy, logo mark, nav link hover underline, filled auth submit button |
| Muted text | `--color-ink-muted` | `#6b6b6b` | Secondary body copy, footer disclaimer text, input placeholder, helper text |
| Destructive | `--color-destructive` | `#8b0000` (deep oxblood) | Error messages on auth modal ONLY (e.g. "That email didn't work. Try again.") |

**Accent (`--color-ink`) is reserved for EXACTLY these elements in Phase 1:**
1. All text (headings, body, labels) ‚Äî ink is the primary readable color on paper
2. Primary CTA fill: "Sign in" button inside AuthModal
3. Primary CTA fill: "Sign out" button on `/account` stub
4. Hamburger icon lines
5. Logo wordmark "Rule 257" in SiteHeader
6. Nav link text (default state)
7. 1px underline on hover/focus for text nav links

**Accent is NOT used for:**
- Input borders (use `--color-hairline`)
- Divider lines (use `--color-hairline`)
- Placeholder text (use `--color-ink-muted`)
- Decorative borders

**Destructive (`--color-destructive`) is reserved for EXACTLY:**
- Error text beneath the auth modal email input
- No destructive buttons in Phase 1 (no delete actions exist yet)

**No second accent color.** No blue links, no green success, no amber warning. Success state in Phase 1 = plain ink text ("Check your email"). The gallery/runway aesthetic depends on this restraint.

**CSS variables (from RESEARCH.md Pattern 2, Phase 1 establishes, Phase 5 inverts for dark mode):**
```css
@theme {
  --color-paper: #fafafa;
  --color-paper-muted: #eaeaea;
  --color-hairline: #e5e5e5;
  --color-ink: #0a0a0a;
  --color-ink-muted: #6b6b6b;
  --color-destructive: #8b0000;
}
```

**Contrast verification (WCAG AA):**
- `#0a0a0a` on `#fafafa`: contrast ratio ‚âà 20.3:1 (AAA pass ‚Äî large and small text)
- `#6b6b6b` on `#fafafa`: contrast ratio ‚âà 5.6:1 (AA pass ‚Äî normal text)
- `#fafafa` on `#0a0a0a` (button label): ‚âà 20.3:1 (AAA pass)
- `#8b0000` on `#fafafa`: ‚âà 9.4:1 (AAA pass)

| Magic link email footer | `This link expires in 10 minutes. If you didn't request it, you can safely ignore this email.` |
| Magic link email address line | `Rule 257 ¬∑ 54 Eldridge St, New York, NY` |

**Voice rules:**
- Lowercase sentence-case for body and UI copy. No Title Case On Every Word.
- No exclamation marks anywhere in Phase 1. Editorial restraint.
- Primary CTAs use verb + noun. `Send magic link`, `Sign out`, `Sign in to Rule 257` ‚Äî never `Submit`, `OK`, `Click here`, `Go`.
- Success messages confirm what just happened and what's next. `Check your email` (what to do) + `We sent a sign-in link to {email}. It expires in 10 minutes.` (what happened + constraint).
- Error messages state the problem + the recovery action in one sentence. `We couldn't send that link. Check the email and try again.`
- Never say `Please`, never say `Oops`, never say `Sorry`.

**Destructive actions in Phase 1:** None. `Sign out` is not destructive ‚Äî it preserves data and can be reversed by signing back in. No confirmation dialog needed. Real destructive actions (redeem reward, adjust points) arrive in Phase 3.

**Empty states in Phase 1:** Not applicable. Phase 1 has no data-driven lists. The Coming Soon page is not an empty state ‚Äî it is the designed content for this phase. Do not render "nothing here yet" language anywhere.

## Interaction & Motion Contract

Phase 1 deliberately locks down motion primitives so Phase 5 (POLISH-02) can layer scroll-driven animation on top without conflicting.

| Interaction | Phase 1 Behavior | Phase Owner |
|-------------|------------------|-------------|
| Header scroll behavior (D-13) | Top bar fades opacity from `bg-paper` to `bg-paper/80` with 8px backdrop-blur as scrollY crosses 24px. No shrink in Phase 1 ‚Äî pure opacity + blur. | Phase 1 |
| Hamburger open | Slide-in from right at `200ms ease-out`. Full viewport overlay on mobile, dismissible via X button or Escape key. | Phase 1 |
| Nav link hover (desktop) | 1px underline draws in from left over `150ms ease-out`. No color change. | Phase 1 |
| CTA button hover | Background opacity shifts from `1.0` to `0.85` over `150ms ease-out`. No scale, no shadow. | Phase 1 |
| Auth modal enter | Fade in backdrop (200ms) + fade-up modal card (300ms ease-out, 8px translate). | Phase 1 |
| Auth modal exit | Reverse of enter (200ms). | Phase 1 |
| Auth modal state transition (input ‚Üí success) | Fade out old content, fade in new content (150ms each, sequential). No layout shift ‚Äî modal height holds. | Phase 1 |
| Form submit spinner | Button label swaps to `Sending...`, button disabled. No spinner icon in Phase 1 ‚Äî text-only state. | Phase 1 |
| Scroll-driven reveals on content | **NOT IMPLEMENTED in Phase 1.** Reserved for Phase 5 POLISH-02. | Phase 5 |
| Parallax on hero imagery | **NOT IMPLEMENTED in Phase 1.** Reserved for Phase 5 POLISH-02. | Phase 5 |
| Haptic feedback on native | **NOT IMPLEMENTED in Phase 1.** Reserved for Phase 4 MOBILE-01 via `@capacitor/haptics`. | Phase 4 |

**Motion tokens:**
```css
@theme {
  --duration-fast: 150ms;
  --duration-base: 200ms;
  --duration-slow: 300ms;
  --ease-out: cubic-bezier(0.16, 1, 0.3, 1);
}
```

**Reduced motion:** All transitions MUST be wrapped in `@media (prefers-reduced-motion: reduce) { * { transition-duration: 0.01ms !important; } }`. Non-negotiable WCAG.

## Accessibility Contract

| Requirement | Phase 1 Implementation |
|-------------|-----------------------|
| Keyboard navigation | All interactive elements reachable via Tab. Auth modal traps focus. Escape dismisses modal + mobile menu. |
| Focus indicators | `2px solid var(--color-ink)` outline with `2px offset`, applied globally via `:focus-visible`. Never removed. |
| Minimum touch target | 44√ó44px on all interactive elements (iOS HIG / WCAG 2.5.5) |
| Color contrast | AA for all text, AAA for primary body+heading (validated above) |
| `prefers-reduced-motion` | Honored globally via `@media (prefers-reduced-motion: reduce)` rule |
| Form labels | Visible `<label>` for email input (sr-only is NOT acceptable ‚Äî the brand wants the word "Email" visible) |
| Landmark roles | `<header>`, `<main>`, `<footer>` semantic tags. Modal uses `role="dialog" aria-modal="true" aria-labelledby` |
| Error association | Auth error message uses `aria-describedby` linked to email input |
| Screen reader announcements | Auth modal state changes use `aria-live="polite"` on the status region |
